Docker compose vite+express

Frontend: react vite nginx

Backend: node express mongo Redis

使用docker compose启动多服务

Docker Compose 文件

FrontEnd

  1. Dockerfile
FROM node:18-alpine3.17 as build

ENV VITE_API_URL=http://backend:8009
ENV NODE_OPTIONS=--max_old_space_size=4096
ENV VITE_DOCKER=true

WORKDIR /app

# Build in docker
COPY package*.json  ./

RUN npm install

COPY . .

RUN npm run build


FROM nginx as production

COPY --from=build /app/dist /usr/share/nginx/html

COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
  1. nginx.conf
events {

}

http {
    sendfile on;
		tcp_nopush on;
    types_hash_max_size 2048;
    # server_tokens off;

    # server_names_hash_bucket_size 64;
    # server_name_in_redirect off;

    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    gzip on;


    server {
        listen       80;
        server_name  localhost;
        location /service/{
            proxy_pass http://backend:8009/;
        }
        location / {
            root   /usr/share/nginx/html/;
            index  index.html;
            try_files $uri /index.html;
        }
    }
}
  1. vite.config.ts
import legacy from "@vitejs/plugin-legacy";
import react from "@vitejs/plugin-react-swc";
import { defineConfig, loadEnv } from "vite";

// https://vitejs.dev/config/
export default ({mode}) => defineConfig({
  plugins: [react(), legacy()],
  test: {
    globals: true,
    environment: "jsdom",
    setupFiles: "./src/setupTests.ts",
  },
  resolve: {
    alias: {
      "@": "/src",
    },
  },
  optimizeDeps: {
    include: ["@mui/icons-material"],
  },
  server: {
    proxy: {
      "/service": {
        target: loadEnv(mode, process.cwd()).VITE_API_URL,
        secure: false,
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/service/, ""), // 不可以省略rewrite
      },
    },
  },
});

BackEnd

Dockerfile

FROM ubuntu:latest

RUN apt-get update && apt-get install -y curl
RUN curl -sL https://deb.nodesource.com/setup_20.x | bash -
RUN apt-get install -y nodejs

RUN apt-get install -y build-essential libcairo2-dev \
 libpango1.0-dev libjpeg-dev libgif-dev librsvg2-dev

ENV MONGO_URL=mongodb://db:27017/db_name
ENV REDIS_HOST=redis

WORKDIR /app

COPY package*.json  ./

RUN npm install

COPY . .

EXPOSE 8009

CMD ["npm", "start"]

因为后端使用了node-canvas，需要安装依赖项，各个操作系统需要的依赖不同，所以指定为Ubuntu系统。

docker-compose.yml

services:
  frontend:
    deploy:
      resources:
        limits:
          memory: 4G
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "5555:80"
    depends_on:
      - backend
    # 设置环境变量，因为vite build的时候需要设置space size，不然会oom
    environment:
      - NODE_OPTIONS=--max-old-space-size=4096
    networks:
      - mynetwork
  backend:
    build:
      context: ./server
      dockerfile: Dockerfile
    ports:
      - "8009:8009"
    depends_on:
      - db
      - redis
    networks:
      - mynetwork
  db:
    image: mongo:4.0.9
    ports:
      - "27017:27017"
    restart: always
    volumes:
      - dbdata:/data/db 	# 映射到卷中，保证删除container数据库文件不会丢失
    networks:
      - mynetwork
  redis:
    image: redis:5.0.5
    ports:
      - "6379:6379"
    networks:
      - mynetwork
# 创建网络，容器间共享网络
networks:
  mynetwork:
    driver: bridge
volumes:
  dbdata:

Docker 运行

  • 运行容器
docker-compose build & docker-compose up
  • 浏览器运行
localhost:5555
  • 如果只想build frontend：
docker-compose build frontend
  • 进入到容器内操作：
docker exec -it container_id /bin/bash
  • 查看当前运行的容器：
docker ps

Docker 私有仓库

安装运行 docker-registry

$ docker run -d -p 5000:5000 --restart=always --name registry registry

在私有仓库上传、搜索、下载镜像

创建好私有仓库之后，就可以使用 docker tag 来标记一个镜像，然后推送它到仓库。例如私有仓库地址为 127.0.0.1:5000。

先在本机查看已有的镜像。

$ docker image ls
REPOSITORY                        TAG                 IMAGE ID            CREATED             VIRTUAL SIZE
ubuntu                            latest              ba5877dc9bec        6 weeks ago         192.7 MB

使用 docker tag 将 ubuntu:latest 这个镜像标记为 127.0.0.1:5000/ubuntu:latest。

格式为 docker tag IMAGE[:TAG] [REGISTRY_HOST[:REGISTRY_PORT]/]REPOSITORY[:TAG]。

$ docker tag ubuntu:latest 127.0.0.1:5000/ubuntu:latest
$ docker image ls
REPOSITORY                        TAG                 IMAGE ID            CREATED             VIRTUAL SIZE
ubuntu                            latest              ba5877dc9bec        6 weeks ago         192.7 MB
127.0.0.1:5000/ubuntu:latest      latest              ba5877dc9bec        6 weeks ago         192.7 MB

使用 docker push 上传标记的镜像。

$ docker push 127.0.0.1:5000/ubuntu:latest

配置非 https 仓库地址

因为 Docker 默认不允许非 HTTPS 方式推送镜像。我们可以通过 Docker 的配置选项来取消这个限制.

Ubuntu 16.04+, Debian 8+, centos 7

对于使用 systemd 的系统，请在 /etc/docker/daemon.json 中写入如下内容（如果文件不存在请新建该文件）

{
  "registry-mirrors": [
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ],
  "insecure-registries": [
    "192.168.199.100:5000"
  ]
}
